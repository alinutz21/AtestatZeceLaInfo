Căutarea Binara

Introducere în Căutarea Binara

Algoritmul de căutare binară este un exemplu clasic de algoritm cu complexitate O(log2(n)). Acesta este utilizat pe vectori ordonați pentru:

  • Determinarea dacă un element există în vector.
  • Găsirea poziției unui element.
  • Calcularea poziției unui element minim sau maxim conform anumitor criterii.
  • Numărarea aparițiilor unui element.
  • Determinarea rădăcinii pătrate sau a părții întregi a radicalului.

1. Verificarea Existenței

Cod Iterativ:

bool cautareBinara(int a[], int n, int val){
    int st = 1, dr = n;
    while(st <= dr){
        int mij = (st + dr) / 2;
        if(a[mij] == val)
            return true;
        else if(a[mij] > val)
            dr = mij - 1;
        else st = mij + 1;
    }
    return false;
}

Cod Recursiv:

bool cautareaBinaraRec(int a[], int st, int dr, int val){
    if(st > dr)
        return false;
    int mij = (st + dr) / 2;
    if(a[mij] == val)
        return true;
    else if(a[mij] > val)
        return cautareaBinaraRec(a, st, mij-1, val);
    return cautareaBinaraRec(a, mij+1, dr, val);
}

2. Poziția unui Element

Cod Iterativ:

int cautareBinaraPozitie(int a[], int n, int val){
    int st = 1, dr = n;
    while(st <= dr){
        int mij = (st + dr) / 2;
        if(a[mij] == val)
            return mij;
        else if(a[mij] < val)
            st = mij + 1;
        else dr = mij - 1;
    }
    return -1;
}

3. Rădăcina unui Număr

Cod Iterativ:

int determinareRadacina(int n){
    int st = 1, dr = n;
    while(st <= dr){
        int mij = (st + dr) / 2;
        if(mij * mij == n)
            return mij;
        else if(mij * mij > n)
            dr = mij - 1;
        else st = mij + 1;
    }
    return -1;
}

Cod Recursiv:

int determinareRadacinaRec(int st, int dr, int n){
    if(st > dr)
        return -1;
    int mij = (st + dr) / 2;
    if(mij * mij == n)
        return mij;
    else if(mij * mij > n)
        return determinareRadacinaRec(st, mij-1, n);
    return determinareRadacinaRec(mij+1, dr, n);
}

4. Observații

  • Căutarea binară funcționează eficient doar pe vectori ordonați.
  • Implementările iterative sunt mai rapide în practică datorită evitării utilizării stivei de apeluri.
  • Complexitatea logaritmică face algoritmul extrem de eficient pentru seturi mari de date.
